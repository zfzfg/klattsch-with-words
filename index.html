bN b=Nabsolute base F0 in Hz (also accepts notes: bC4, bC#5, bDb3, bA-1, etc.)
b+N b-Nrelative base F0 (running pitch shifts by N)

f0 evolution

  • stress (!) is transient: doesn't carry forward
  • bare pitch deltas +N/-N are sticky: mutates the running pitch
  • parenthesized deltas (+N)/(-N) are transient: this phoneme only
  • bN (or bC4 etc.) is an absolute reset to that pitch
  • bare b returns to the utterance's starting pitch

changelog

v0.5.0

  • complete UI overhaul: instrument-panel look, proper mobile layout, same controls
  • syntax highlighting in the editor: phonemes, directives, pitch marks, comments; unknown phonemes flagged as you type
  • community showcase: a gallery of things people made, different feature each visit

v0.4.0

  • phoneme banks: layered tables with per-call selection and a UI dropdown above the audition row
  • first non-English bank: japanese vowels and the alveolar tap, with two tunings (.hecko on discord, Mokhtari & Tanaka 2000 corpus means)
  • mid-utterance [bank=NAME] directive switches active bank for subsequent phonemes; bare [bank] resets

v0.3.0

  • karaoke playback: live display under the textarea highlights the active phrase as audio runs
  • video render: phrase teleprompter with smooth vertical scroll
  • video readability: subtitle back gradient, text drop shadow, highlight rect outline
  • optional karaoke highlight toggle on the video render
  • stop button cancels live audition mid-utterance
  • parser bugfix: pitch bends now accept decimals (AY+1.5, OW(-0.25))
  • this changelog popup

v0.2.0

  • tremolo: m depth and n rate directives, plus output volume slider
  • audition buttons get example words with the relevant letters underlined
  • live slider state mirror, with an insert button to prepend as directives
  • video render: custom title in the upper-left, fade gradient at the top
  • render-cancel button, spacebar guard
  • custom export filename input
  • /* */ block comments
  • Unicode normalization: homoglyphs to Latin, zero-width chars stripped
  • localStorage autosave for the textarea
  • WAV files embed the source utterance string in the RIFF INFO chunk

v0.1.2

  • engine: state reset on each play, no carry-over between utterances
  • engine: glottal phase wrap fix (the 374 GHz bF34 situation)

v0.1.1

  • npm package fixes (bin name matches package name)

v0.1.0

  • initial release: parallel-formant engine, ARPABET phoneme strings, directive language, WAV/video export
